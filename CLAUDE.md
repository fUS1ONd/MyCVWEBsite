# Техническое задание: Персональный веб-сайт (CV + AI Blog)

## 1. Общее описание проекта

Персональный веб-сайт, выполняющий две основные функции:

1. **CV-страница** для потенциальных работодателей (публичный доступ)
2. **AI Blog** с постами об искусственном интеллекте (для авторизованных пользователей)

---

## 2. Функциональные требования

### 2.1. Для неавторизованных пользователей (публичная часть)

Главная страница должна содержать:

- **О себе** - описание личности, навыков, опыта
- **О деятельности** - текущие занятия, направления работы
- **Фотография** - профессиональное фото
- **Контакты** - способы связи (email, социальные сети, etc.)

**Ограничения:**

- Просмотр контента ограничен только CV-информацией
- Нет доступа к постам и комментариям

### 2.2. Для авторизованных пользователей

Дополнительно к публичной части:

#### 2.2.1. Просмотр постов об AI

- Лента постов с текстом, изображениями и видео
- Форматированный текст (жирный, курсив, заголовки, списки, ссылки)
- Встраивание изображений в посты
- Встраивание видео в посты
- Поддержка markdown или rich text редактора

#### 2.2.2. Комментарии

- Возможность комментировать посты автора
- Отображение имени и аватара пользователя
- Временные метки комментариев

### 2.3. Система авторизации

**Поддерживаемые провайдеры OAuth:**

- ВКонтакте (VK)
- Google
- GitHub

**Требования:**

- Безопасное хранение токенов
- Автоматическое обновление сессии
- Возможность выхода из системы

### 2.4. Система уведомлений

**Для авторизованных пользователей:**

- Уведомления о новых постах
- Способы доставки: email и/или push-уведомления в браузере
- Возможность подписки/отписки от уведомлений

### 2.5. Административная панель (автор сайта)

**Управление контентом:**

- Создание новых постов
- Редактирование существующих постов
- Удаление постов
- Rich text редактор для форматирования
- Загрузка изображений и видео
- Модерация комментариев

---

## 3. Нефункциональные требования

### 3.1. Дизайн и UX

- Современный, чистый и понятный интерфейс
- Профессиональный внешний вид для CV-части
- Адаптивная верстка (responsive design)
- Полная поддержка мобильных устройств
- Поддержка планшетов и десктопов
- Оптимизация для различных размеров экранов

### 3.2. Производительность

- Быстрая загрузка страниц (< 3 секунд)
- Оптимизация изображений
- Lazy loading для медиа-контента
- Кэширование статических ресурсов

### 3.3. SEO и аналитика

- Интеграция с Яндекс.Метрикой
- Интеграция с Google Analytics
- Правильные meta-теги для SEO
- Open Graph разметка для социальных сетей
- Sitemap.xml
- Robots.txt

### 3.4. Безопасность

- HTTPS (SSL-сертификат)
- Защита от CSRF атак
- Защита от XSS атак
- Безопасное хранение паролей и токенов
- Rate limiting для API
- Валидация всех пользовательских вводов

### 3.5. Тестирование

Должны быть покрыты тестами:

- **Unit тесты** - бизнес-логика, утилиты, функции
- **Integration тесты** - API endpoints, взаимодействие с БД
- **E2E тесты** - критические пользовательские сценарии

**Цели покрытия:**

- Backend: минимум 70% покрытия кода
- Frontend: минимум 60% покрытия компонентов

---

## 4. Метрики успеха

- Сайт загружается за < 3 секунд
- Адаптивная верстка работает на всех устройствах
- SSL работает корректно (A+ рейтинг на SSL Labs)
- Авторизация через все 3 провайдера работает
- Покрытие тестами > 70% для backend
- Нет критических уязвимостей безопасности
- Метрики отслеживаются корректно
